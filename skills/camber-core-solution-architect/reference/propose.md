# PROPOSE — the full framework

Goal: surface where Camber Core has the most impact for this company, ranked, and let
the person steer.

## The opportunity table

Present a prioritized table. One row per opportunity. Keep it specific to their
departments and named systems.

| # | Opportunity | Department | Manual work it removes | Camber Core layer(s) | Impact | Effort |
|---|---|---|---|---|---|---|
| 1 | Cross-portco pipeline review in one thread | Deal team / ops | Logging into N CRMs, stitching a deck | Isolation + Access | High | Low |
| 2 | Skill capture for portco onboarding | Ops | Re-teaching the same playbook each time | Capture | High | Med |
| 3 | … | … | … | … | … | … |

After the table, ask in **numbered options** which to pursue:
> Which of these should I design first?
> 1. #1 — cross-portco review (fastest visible win)
> 2. #2 — onboarding skill capture (compounding)
> 3. A different one you care about more
> 4. You're wrong about the ranking — here's why

## The differentiation test (apply to every row)

> "Could they get this from native AI tooling with one credential set?"

If **yes**, it's not a Camber Core opportunity — it's a chatbot over one tool. Cut it or
deepen it. Every opportunity must lean on at least one layer:

- **Isolation** — the value depends on data staying walled per entity.
- **Access** — the value depends on a principal operating *across* entities.
- **Capture** — the value depends on turning real work into reusable skills.
- **Fit** — the value depends on staying current as the business moves.

The strongest opportunities use several at once.

## Surface areas by department (prompts, not a menu to dump)

- **Leadership / principals:** cross-entity rollups, portfolio diligence as a byproduct,
  "ask any entity by name" review sessions.
- **Ops:** consolidation of fragmented systems, standardized workflows across entities,
  onboarding playbooks captured as skills.
- **Sales / RevOps:** unified pipeline view, contact-intelligence reuse, qualification
  logic captured from what actually closes.
- **Marketing:** cross-entity campaign context, governed brand rules applied everywhere.
- **Customer success:** health rollups, scoped client visibility without exposing
  internal systems.
- **Finance / clinical / domain:** roll-up metrics across entities while leaving each
  source system untouched.

Propose only what's relevant to *their* structure and departments. A roll-up cares about
different surfaces than an agency.

**Output:** a ranked opportunity list with clear, differentiated value propositions, and
the person's pick of what to design first.
